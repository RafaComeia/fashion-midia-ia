## Story
**As a** desenvolvedor,
**I want** configurar o monorepo com Next.js 14, NestJS e Turborepo com todas as ferramentas de qualidade,
**so that** todos os módulos compartilhem uma base sólida, padronizada e pronta para desenvolvimento.

## Acceptance Criteria
1. Monorepo configurado com Turborepo contendo `apps/web` (Next.js 14 App Router), `apps/api` (NestJS) e `packages/shared`
2. ESLint configurado globalmente com regras para TypeScript, com script `lint` funcionando via Turborepo
3. Prettier configurado globalmente com `.prettierrc`, com script `format` funcionando
4. TypeScript configurado com `tsconfig.base.json` na raiz e configs estendidas em cada app
5. Scripts de `dev`, `build` e `test` funcionando via Turborepo (`turbo run dev`, `turbo run build`, `turbo run test`)
6. Variáveis de ambiente documentadas em `.env.example` na raiz e em cada app
7. README.md na raiz com instruções completas de setup local (pré-requisitos, instalação, scripts disponíveis)
8. `packages/shared` exporta tipos TypeScript básicos e é importável pelos dois apps sem erros

```
├── package.json      # root (workspaces)

├── .prettierrc
└── README.md
```

### Configuração Turborepo (turbo.json)
```json
{
  "$schema": "https://turbo.build/schema.json",
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**", "dist/**"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "test": {
      "dependsOn": ["build"]
    },
    "lint": {}
  }
}
```

### Package.json root
```json
{
  "name": "fashion-midia-ia",
  "workspaces": ["apps/*", "packages/*"],
  "scripts": {
    "dev": "turbo run dev",
    "build": "turbo run build",
    "test": "turbo run test",
    "lint": "turbo run lint",
    "format": "prettier --write \"**/*.{ts,tsx,js,json,md}\""
  }
}
```
