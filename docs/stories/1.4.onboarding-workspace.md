## Story
**As a** usuário recém cadastrado,
**I want** configurar o workspace da minha marca em um wizard de 3 passos,
**so that** minha empresa esteja isolada e pronta para criar campanhas na plataforma.

## Acceptance Criteria
1. Wizard de onboarding com 3 passos sequenciais: (1) Nome da marca, (2) Segmento principal, (3) Upload de logo
2. Passo 1: campo "Nome da marca" obrigatório (2–100 chars) + "Slug" gerado automaticamente (editável, único, URL-safe)
3. Passo 2: seleção do segmento principal via cards visuais com os 8 estilos de moda (casual, festa, fitness, infantil, executivo, praia, noivas, plus size)
4. Passo 3: upload de logo (JPG/PNG/SVG/WebP, máx. 5MB) com preview — opção de pular
5. Ao concluir: workspace criado no banco com `workspace_id` no JWT do usuário (via `user_metadata` do Supabase Auth)
6. Usuário criador associado como `admin` do workspace em `workspace_members`
7. Após conclusão do onboarding, redirecionar para `/dashboard`
8. Opção "Pular e completar depois" disponível nos passos 2 e 3 (passo 1 é obrigatório)
9. Se usuário acessar `/onboarding` com workspace já criado, redirecionar para `/dashboard`

## Dev Notes

### Fluxo de criação do workspace
```
Usuário preenche wizard (3 passos)
         │
         ▼
POST /workspaces (NestJS)
         │
         ├─→ Prisma transaction:
         │     INSERT INTO workspaces (...)
         │     INSERT INTO workspace_members (role: 'admin')
         │
         ├─→ Supabase Admin SDK:
         │     updateUser({ user_metadata: { workspace_id } })
         │     (isso renova o JWT com o workspace_id embutido)
         │
         └─→ Response: { workspace }
                 │
                 ▼
         Frontend: refresh da sessão → redirect /dashboard
```

### Geração automática de slug
```typescript
function generateSlug(name: string): string {
  return name
    .toLowerCase()
    .normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '') // remove acentos
    .replace(/[^a-z0-9\s-]/g, '')    // remove caracteres especiais
    .replace(/\s+/g, '-')            // espaços → hífens
    .replace(/-+/g, '-')             // múltiplos hífens → um
    .trim();
}
```

### Atualizar workspace_id no JWT (Supabase Admin SDK)
```typescript
// No workspaces.service.ts, após criar o workspace:
const supabaseAdmin = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY);
await supabaseAdmin.auth.admin.updateUserById(userId, {
  user_metadata: { workspace_id: workspace.id }
});
```

### Estrutura da tabela workspaces (do schema)
```sql
workspaces: id, name, slug (unique), logo_url, segment, plan, is_active
workspace_members: workspace_id, user_id, role ('admin'|'editor'|'viewer')
```
