## Story
**As a** desenvolvedor,
**I want** um pipeline de CI/CD configurado com deploy automático no Vercel e Railway,
**so that** cada merge na main faça deploy automático e a plataforma esteja acessível online com health-check funcional.

## Acceptance Criteria
1. GitHub Actions configurado com job de CI: lint + typecheck + build + testes automáticos em todo PR
2. Deploy automático do frontend (`apps/web`) na Vercel ao merge na branch `main`
3. Deploy automático do backend (`apps/api`) no Railway ao merge na branch `main`
4. Ambiente de staging separado do produção (branch `develop` → staging, `main` → produção)
5. Health-check endpoint `GET /health` no backend retornando `{ status: "ok", timestamp: "<ISO>" }` com HTTP 200
6. Variáveis de ambiente configuradas nos dois ambientes (staging e produção) no Vercel e Railway
7. Badge de status do CI no README.md

## Dev Notes

### Estrutura dos workflows
```
.github/
└── workflows/
    ├── ci.yml              — lint + build + test (todo PR)
    ├── deploy-staging.yml  — deploy para staging (merge em develop)
    └── deploy-production.yml — deploy para produção (merge em main)
```

### Exemplo de ci.yml
```yaml
name: CI
on:
  push:
    branches: ['**']
  pull_request:
    branches: [main, develop]

jobs:
  ci:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
      - run: npm ci
      - run: turbo run lint typecheck build test
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ vars.TURBO_TEAM }}
```
