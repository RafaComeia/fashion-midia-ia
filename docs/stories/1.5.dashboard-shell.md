## Story
**As a** usuário autenticado com workspace criado,
**I want** visualizar o dashboard principal com navegação, KPIs e atalho para criar campanha,
**so that** tenha uma visão geral da plataforma e acesso rápido a todas as funcionalidades.

## Acceptance Criteria
1. Layout principal responsivo com sidebar de navegação contendo: Campanhas, Performance, Capacitação e Configurações
2. Header com logo da plataforma (Fashion Mídia.IA), nome/avatar do usuário e menu dropdown com opção de logout
3. Área de boas-vindas exibindo o nome da marca (workspace) e botão "Criar primeira campanha" (destaque visual)
4. Cards de KPIs na área principal: Total de Campanhas, Total de Assets Gerados, Engajamento Médio — todos zerados com tooltip explicativo
5. Sidebar responsiva: expanded no desktop (≥ 1024px), colapsada com ícones no tablet (768–1023px), drawer no mobile (< 768px)
6. Rota `/dashboard` protegida — usuário sem workspace redireciona para `/onboarding`, sem autenticação para `/login`
7. Navegação ativa destacada na sidebar conforme a rota atual (active state)
8. Logout funcional: encerra sessão Supabase e redireciona para `/login`
9. Plataforma responsiva em resolução mínima de 375px (mobile) até 1440px (desktop)

### Hook useWorkspace
```typescript
// hooks/useWorkspace.ts
export function useWorkspace() {
  const supabase = createClient()
  return useQuery({
    queryKey: ['workspace'],
    queryFn: async () => {
      const { data: { user } } = await supabase.auth.getUser()
      const workspaceId = user?.user_metadata?.workspace_id
      if (!workspaceId) return null
      // buscar workspace via API ou Supabase direto
      const { data } = await supabase
        .from('workspaces')
        .select('*')
        .eq('id', workspaceId)
        .single()
      return data
    }
  })
}
```
